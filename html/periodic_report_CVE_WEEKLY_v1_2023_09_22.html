Chromium: CVE-2023-4909 Inappropriate implementation in Interstitials

Revision: 1.0 (NEW)
Published: 15-09-2023
Official Fix:
Yes
Product family info: microsoft edge (chromium-based)
Show More
Build numbers:
117.0.2045.31
KB Articles:
  • Release Note (15-09-2023)
Update Packages:
  • Edge Update
Summary: Microsoft Edge (Chromium-based) is affected by CVE-2023-4909, an inappropriate implementation in Interstitials, assigned by Chrome. This vulnerability is in the Chromium Open Source Software (OSS) consumed by Microsoft Edge. An attacker could exploit this vulnerability to execute arbitrary code in the context of the current user. Microsoft has addressed this vulnerability by ingesting Chromium, and the latest version of Microsoft Edge (Chromium-based) is no longer vulnerable. System administrators are advised to update Microsoft Edge to the latest version. For more information, refer to the Google Chrome Releases page. The vulnerability affects Microsoft Edge (Chromium-based) with build number 117.0.2045.31. Microsoft has provided a security update to mitigate this vulnerability. It is important for system administrators to stay informed about the support lifecycle for their software by referring to the Microsoft Support Lifecycle.

Summary: Microsoft Edge (Chromium-based) is affected by an elevation of privilege vulnerability, identified as CVE-2023-36735, with a CVSS score of 9.6/8.3. The vulnerability allows an attacker to gain unauthorized access and execute arbitrary code. It is rated as moderate severity due to the level of user interaction required for exploitation. Summary: Windows MSHTML Platform Security Feature Bypass Vulnerability, identified as CVE-2023-36805, impacts all supported versions of Microsoft Windows. The vulnerability allows for remote code execution with a base score of 7.0 and a temporal score of 6.1. It is caused by a flaw in the MSHTML platform, which could lead to high privilege access for an attacker. Successful exploitation could result in a major loss of confidentiality, integrity, and availability. Microsoft recommends installing the September 2023 updates to be fully protected. Customers who install the Security Only updates for affected Windows Server versions should also install the IE Cumulative update. Automatic update configurations do not need further action. The vulnerability affects Windows Server 2008 R2, Windows Server 2012, and other Windows versions. The underlying MSHTML, EdgeHTML, and scripting platforms are still supported despite the retirement of Internet Explorer 11 and deprecation of Microsoft Edge Legacy. The IE Cumulative updates include changes to address vulnerabilities in the MSHTML platform and scripting engine. System administrators should prioritize the installation of the recommended updates to mitigate the risk of exploitation.
