Summary: Windows OLE Remote Code Execution Vulnerability (CVE-2023-35323) has been updated with an acknowledgment, indicating an informational change only. Summary: Microsoft Message Queuing is affected by a critical remote code execution vulnerability, identified as CVE-2023-32057. The vulnerability allows an attacker to send a specially crafted malicious MSMQ packet to a MSMQ server, resulting in remote code execution on the server side. The attack vector is network-based, with low attack complexity and no privileges required. The vulnerability impacts confidentiality, integrity, and availability, leading to a total loss in each aspect. Microsoft has provided an official fix for this vulnerability through security updates for various Windows Server versions, Windows 10, and Windows 11. System administrators are advised to apply the respective security updates to mitigate the risk of exploitation. Additionally, a mitigating factor is available, where the Windows message queuing service needs to be enabled for a system to be exploitable by this vulnerability, and this feature can be added via the Control Panel.

Summary: Windows Bluetooth A2DP driver is affected by an elevation of privilege vulnerability, identified as CVE-2023-35387. The vulnerability allows an authorized attacker to programmatically run certain functions, leading to elevation of privilege on the Bluetooth component, potentially gaining SYSTEM privileges. Successful exploitation requires the victim to pair with the attacker's Bluetooth device. The attack vector is adjacent, requiring the attacker to gain access to the restricted network before running an attack. The vulnerability affects various Windows versions, including Windows 10, 11, and Windows Server 2012, 2016, and 2019. Microsoft has provided a security update to address this vulnerability, and system administrators are advised to apply the relevant security updates to mitigate the risk of exploitation.
