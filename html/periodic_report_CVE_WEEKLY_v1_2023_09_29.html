Chromium: CVE-2023-5217 Heap buffer overflow in vp8 encoding in libvpx

Revision: 1.0 (NEW)
Published: 29-09-2023
Official Fix:
Yes
Product family info: microsoft edge (chromium-based)
Show More
Build numbers:
116.0.1938.98
KB Articles:
  • Release Note (29-09-2023)
Update Packages:
  • Edge Update
Summary: Microsoft Edge (Chromium-based) is affected by a heap buffer overflow vulnerability in vp8 encoding in libvpx, identified as CVE-2023-5217. This vulnerability, assigned by Chrome, is addressed by ingesting Chromium. An exploit for CVE-2023-5217 is known to exist in the wild. The vulnerability is caused by a flaw in the Chromium Open Source Software (OSS) consumed by Microsoft Edge (Chromium-based). Attackers could exploit this vulnerability to execute arbitrary code. Microsoft has released a security update for Microsoft Edge (Chromium-based) with the build number 116.0.1938.98, making the latest version no longer vulnerable. Microsoft is also working to identify and address the impact of this vulnerability on Microsoft Teams and Skype applications. System administrators are advised to update their Microsoft Edge installations to the latest version to mitigate the risk of exploitation. For more information, refer to the Google Chrome Releases page.

CVE-2023-38148 Internet Connection Sharing (ICS) Remote Code Execution Vulnerability

Revision: 1.1
Published: 28-09-2023

Information update only
Product family info: windows 10 21H2 32-bit_systems, windows 10 21H2 x64-based_systems, windows 10 22H2 32-bit_systems, windows 10 22H2 x64-based_systems, windows 11 21H2 x64-based_systems, windows 11 22H2 x64-based_systems

KB Articles:

Update Packages:

  • Security Update (KB5030217)

Summary: Internet Connection Sharing (ICS) is affected by a Critical Remote Code Execution vulnerability, CVE-2023-38148. The vulnerability allows an unauthorized attacker to send a specially crafted network packet to the ICS service, leading to remote code execution. The attack vector is adjacent, meaning it is limited to systems connected to the same network segment as the attacker. No user interaction or privileges are required for exploitation. The impact includes high confidentiality, integrity, and availability loss. Microsoft has released security updates for various affected products, including Windows 10, Windows 11, and Windows Server 2022, addressing the vulnerability. System administrators are advised to apply the official fix to mitigate the risk of exploitation. Exploitation of this vulnerability requires that the Internet Connection Sharing (ICS) is enabled, and no exploit code is currently available. For more information, refer to the Microsoft Security Support Lifecycle.

KB Articles:
  • Release Note (02-06-2023)
Update Packages:
  • Edge Update
Summary: Microsoft Edge (Chromium-based) is affected by a Security Feature Bypass vulnerability, CVE-2023-29345, with a CVSS score of 6.1/5.3. The vulnerability allows an attacker to bypass the permissions dialog feature prompt presented to users when initiating a download, requiring user interaction by clicking on a specially crafted URL. Successful exploitation could lead to a browser sandbox escape, impacting integrity and confidentiality. The attack vector is network-based, with low attack complexity and no privileges required. Microsoft has released a security update (build number 114.0.1823.37) to address this vulnerability. System administrators are advised to apply the official fix to mitigate the risk of exploitation. For more information, refer to the Microsoft Security Support Lifecycle.

Update Packages:

  • Security Update (KB5030219)

Summary: Windows MSHTML Platform Security Feature Bypass Vulnerability, identified as CVE-2023-36805, impacts all supported versions of Microsoft Windows. The vulnerability allows for remote code execution with a base score of 7.0 and a temporal score of 6.1. It is caused by a flaw in the MSHTML platform, which could lead to high privilege access for an attacker. Successful exploitation could result in a major loss of confidentiality, integrity, and availability. Microsoft recommends installing the September 2023 updates to be fully protected. Customers who install the Security Only updates for affected Windows Server versions should also install the IE Cumulative update. Automatic update configurations do not need further action. The vulnerability affects Windows Server 2008 R2, Windows Server 2012, and other Windows versions. The underlying MSHTML, EdgeHTML, and scripting platforms are still supported despite the retirement of Internet Explorer 11 and deprecation of Microsoft Edge Legacy. The IE Cumulative updates include changes to address vulnerabilities in the MSHTML platform and scripting engine. System administrators should prioritize the installation of the recommended updates to mitigate the risk of exploitation.
